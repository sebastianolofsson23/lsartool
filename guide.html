Getting Started

LSAR Tool is a portable Windows application — no installation required. Download the zip, extract it anywhere, and run LSAR Tool.exe.

When you first open the app you'll land on the Dashboard. Your data is stored in a local .db file on your computer. Everything works offline.

Tip: On first launch Windows may show a SmartScreen warning. Click "More info" then "Run anyway" — this is normal for unsigned apps.

Projects

Each project is its own isolated database file. You can have multiple projects open in the sidebar and switch between them instantly.

Creating a project

Click the + button in the Projects section of the sidebar. Give it a name — the database file will be saved as ProjectName.db in your chosen storage folder.

Opening an existing project

Click the 📂 button to browse for an existing .db file. Useful for opening databases from a shared network drive.

Renaming & closing

Each project in the sidebar has a ✏️ rename button and a ✕ close button. Closing removes it from the list but does not delete the file.

Storage location

Click 📁 Storage at the bottom of the Projects panel to choose where new database files are saved.

Item Catalogue

The catalogue is your master list of all maintainable hardware items. Every item in the LCN hierarchy must reference a catalogue entry.

Adding items

Click + Add Item and fill in the part number, name, category and failure rate. Failure rate is in Failures per Million Hours (FPMH).

Inline editing

Click any cell in the table to edit it directly. Press Tab to move to the next cell, Enter to confirm. Changes save automatically on blur.

Tip: Use the column filter buttons (▾) in each header to filter and sort the table.

LCN Hierarchy

The LCN (Logistic Control Number) Hierarchy defines the hardware breakdown structure of your system — equivalent to LSA Table 010 in S3000L.

Adding nodes

Click + Add Node. Each node requires a unique LCN code and a reference to a catalogue item. You can set a parent LCN to build the tree structure.

Tree view

Nodes are displayed in a collapsible tree. Click the ▶ toggle to expand/collapse branches.

Tip: LCN codes are free-form text — use any convention that fits your programme (e.g. A, A-1, A-1-1 or 0100, 0101, etc.).

FMECA

The FMECA Register stores all failure modes, effects and criticality data — equivalent to LSA Table 030. System MTBF and MTTR are calculated automatically from this data.

Adding records

Click + Add Record. Select the LCN node, then fill in the function, failure mode, effects chain and severity category.

Severity categories

  • Critical — Loss of life or mission-critical system failure
  • High — Major degradation of system function
  • Medium — Minor degradation, workaround available
  • Low — Cosmetic or negligible impact

Failure Mode Ratio (%)

Each failure mode can be assigned a percentage of all faults for that LCN. The sum across all modes for a given LCN cannot exceed 100%. This feeds into the criticality MTBF cards on the dashboard.

Inline LCN editing

The LCN column in the FMECA table is a dropdown — you can reassign a failure mode to a different LCN node directly in the table.

Maintenance Tasks

Preventive maintenance task analysis — equivalent to LSA Table 040. Tasks are linked to LCN nodes and drive the PM summary on the dashboard.

Adding tasks

Click + Add Task. Select the LCN node, set a task ID, description, interval, man-hours, personnel count and skill level.

Skill levels

  • Basic — Standard maintenance technician
  • Intermediate — Specialist or systems technician

Support equipment

Click the ✏️ button on any task to open the edit modal. The Support Equipment section lets you link required STE to the task with quantities.

Support Equipment

The Support Equipment catalogue stores all Special and Standard tools and equipment — equivalent to LSA Table 050.

Categories

  • Special — Purpose-built or programme-specific tools (e.g. borescope kit, hydraulic test rig)
  • Standard — Common workshop or general tools (e.g. torque wrench, multimeter)

Dashboard

The dashboard gives a live summary of your LSAR data. All values update automatically as you add or edit records.

PM Summary inputs

Set your utilisation profile (operating hours/year, cycles/year, fleet size) to compute fleet-level maintenance burden. These settings are saved per project.

FMECA Coverage

The coverage bar shows the percentage of LCN nodes that have at least one FMECA record. Aim for 100% coverage on all safety-critical items.

Exports

All exports follow the ASD S3000L standard format. Go to the Reports page to export.

Available exports

  • FMEA Register — LSA Table 030 as a formatted Excel workbook
  • Hardware Breakdown — LSA Table 010 as a formatted Excel workbook
  • Full LSAR Workbook — All tables (010–050) in one workbook with a cover sheet
  • Dashboard PDF — PDF summary report of all dashboard metrics

Each export opens a Save As dialog — choose any folder on your computer or network drive.

Import

Import data from a previously exported Full LSAR Workbook (.xlsx). Existing records with the same LCN or part number are skipped — no duplicates.

Note: The import only reads Catalogue, Hardware and FMEA sheets. Maintenance tasks and support equipment must be added manually after import.
